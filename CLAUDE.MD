# 项目说明

love2d 游戏引擎快速开发模板

- workspace: `.`
- platform: win10 / android / ios
- engine: LÖVE 11.5
- language: Lua
- scm: git（已初始化）

---

## 一、权限范围（Permissions）

操作范围：当前项目目录及其所有子目录，拥有完整读写权限。

外部只读权限：
- https://github.com/love2d/love

### 文件操作
- 读取、创建、修改、重命名任意文件/文件夹
- 删除操作需用户明确要求

### Shell 命令
可直接执行，无需额外确认：
- 文件类：ls、cat、rm、mv、cp、mkdir
- 运行类：python、node、lua、bash
- 工具类：grep、sed、awk、curl、wget
- 项目脚本：当前目录下任意脚本

### 3. 工具内置命令和指令
可以直接执行：工具前置指令、内置 skills、内置 agents、plugins

---

## 二、行为规范（Behavior Rules）

- 除非必要，使用中文回复
- 始终假设对整个目录树拥有持续访问权限
- 文件/目录操作请求直接执行，无需额外确认
- Shell 命令直接执行并返回输出
- 路径或命令有歧义时，向用户确认
- 禁止访问项目目录树之外的系统路径（如 `C:\`、`/etc`、`/usr` 等）

---

## 三、Git 规范（Git Rules）

### 1. 初始化
若未初始化 Git 仓库，自动执行 `git init` 并创建初始提交：
若 `.gitignore` 不存在，在首次提交前自动创建。
```
Initial commit - Project initialized
```

### 2. 提交粒度
- 每完成一个功能点/修复点，执行一次提交
- 不允许将无关修改合并到同一提交

### 3. 提交信息格式
```
feature: 描述新增的功能
fix: 描述修复的问题
refactor: 描述代码重构
docs: 描述文档更新
```

### 4. 提交流程
每次完成需求后，必须自动执行：
```
git add .
git commit -m "<符合规范的提交信息>"
```

> 如果是 git 子目录，只提交本目录下内容。

## 四、自动加载文件（Auto Load）

每次会话开始时，自动读取以下文件并作为上下文：

| 文件 | 用途 | 维护者 |
|------|------|--------|
| `memory.md` | 长期记忆，简洁明了 | AI 自行维护 |
| `feedback.md` | 版本变更与问题记录 | 用户 + AI 共同维护 |
| `task.md` | 开发任务与进度 | AI 维护 |

**若文件不存在，自动创建空模板：**

`memory.md` 模板：
```markdown
# 项目记忆
<!-- AI 维护的长期记忆，记录关键决策和项目状态 -->
```

`feedback.md` 模板：
```markdown
# 版本变更记录

| 状态 | 类型 | 编号 | 描述 | 优先级 | 日期 |
|------|------|------|------|--------|------|

## 详细记录
<!-- 记录 BUG 或功能的详细说明：问题描述、修复内容、影响范围 -->
```

`task.md` 模板：
```markdown
# 开发任务

## 待办
<!-- - [ ] 任务描述 -->

## 进行中

## 已完成
```

## 五、工作流程规则（Workflow Rules）

### 1. 任务连续性
- 每次会话检查 `task.md` 中的待办任务
- 有未完成任务时优先继续处理
- 按顺序完成，不跳过

### 2. 代码修改后的文档更新
每次代码修改完成后，必须按以下顺序执行：
1. 读取 `feedback.md` 和 `task.md`
2. 检查是否有后续任务需要继续
3. 更新 `feedback.md`（版本变更表格 + 详细记录）
4. 更新 `task.md`（标记完成状态）
5. Git 提交

> 不允许只修改代码不更新文档。
